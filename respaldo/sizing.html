⚡ CPU e IOPS

Carga de CPU

  • 📊 ~450 operaciones/segundo en pico
  • 🔄 Conexiones concurrentes múltiples
  • 📈 Agregaciones y consultas complejas
  • ✅ 8 vCPUs recomendadas

IOPS Requeridos

  • 💾 Working Set optimizado en RAM
  • ✍️ Escrituras persistentes al disco
  • 🔍 Cache misses ocasionales
  • ✅ Tier "General" suficiente

Arquitectura Final Solo 3 Nodos

🏗️ Diseño Arquitectónico Completo

La siguiente arquitectura muestra la integración completa de MongoDB Atlas con los microservicios de Jugada de Pagos, incluyendo los flujos de datos, componentes de seguridad y la distribución en zonas de disponibilidad.

MongoDB Atlas 8.0

Componentes Clave de la Arquitectura:

📍 Data Layer
  • • MongoDB Atlas M50
  • • 3 nodos Replica Set
  • • Multi-AZ deployment
  • • Auto-failover
🔧 Application Layer
  • • Microservicios Java 21
  • • Framework Quarkus
  • • Kubernetes orchestration
  • • Connection pooling
🔒 Security Layer
  • • TLS/SSL encryption
  • • VPC Peering
  • • IP Whitelisting
  • • RBAC authentication
🚀 Flujos de Datos Principales:
  1. Directorio de Alias: Consultas de lectura intensiva con cache en memoria para respuestas sub-segundo
  2. Procesamiento de Pagos: Escrituras transaccionales con confirmación inmediata y replicación síncrona
  3. Conciliación: Procesos batch programados con agregaciones complejas sobre datos históricos
  4. Análisis en Tiempo Real: Change Streams para eventos y notificaciones instantáneas
