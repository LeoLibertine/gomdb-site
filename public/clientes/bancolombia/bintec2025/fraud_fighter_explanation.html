⚖️ Crear Modelo de Riesgo

Definiendo las reglas del juego

risk_model = {
"weights": {
"amount_anomaly_high": 30,
"location_anomaly": 25,
"new_device": 35,
"unusual_time": 15
},
"thresholds": {
"approve": 30,
"flag": 60,
"block": 85
}
}
db.risk_models.insert_one(risk_model)

🎯 ¿Qué hace?

Crea las reglas fundamentales para decidir si una transacción es fraude:

  • Weights (Pesos) → Cuántos puntos suma cada anomalía detectada
  • Thresholds (Umbrales) → Cuándo aprobar, revisar o bloquear
  • El sistema acumula puntos automáticamente y toma decisiones
Anomalía Detectada Puntos Descripción
Monto muy alto +30 3x el promedio del cliente
Ubicación rara +25 Ciudad diferente a la habitual
Dispositivo nuevo +35 Nunca antes usado
Hora sospechosa +15 Entre 0-6 AM (madrugada)

🚦 Umbrales de Decisión:

✅ APROBAR → Menos de 30 puntos (bajo riesgo)
🟡 REVISAR → Entre 30 y 85 puntos (riesgo medio)
🔴 BLOQUEAR → Más de 85 puntos (alto riesgo)
Analogía:

Es como el sistema de puntos en tu licencia de conducir. Cada infracción suma puntos: exceso de velocidad (+15), pasarse un semáforo (+30), conducir ebrio (+50). Si llegas a 100 puntos, te suspenden la licencia. Aquí funciona igual pero para transacciones financieras.

🧮 Calcular Score de Riesgo

El cerebro que suma puntos sospechosos

def calcular_risk_score(transaccion, cliente, modelo):
score = 0

if monto > promedio * 3:
score += 30 # Monto anómalo

if hora entre 0 y 6:
score += 15 # Hora sospechosa

if dispositivo_nuevo:
score += 35 # Dispositivo desconocido

return score, detalles

⚙️ ¿Qué hace?

Calcula automáticamente cuántos puntos de riesgo tiene una transacción:

  • Comienza con score = 0
  • Revisa cada característica de la transacción
  • Por cada anomalía detectada, suma los puntos correspondientes
  • Al final, compara el total con los umbrales para decidir

🔍 Ejemplo Paso a Paso:

TRANSACCIÓN: $5,000,000 COP a las 3 AM desde iPhone desconocido
PROMEDIO DEL CLIENTE: $100,000 COP
⬇️
+30 puntos → Monto es 50x el promedio
+15 puntos → Hora: 3 AM (madrugada)
+35 puntos → Dispositivo nunca visto
⬇️
SCORE FINAL: 80 puntos
⬇️
DECISIÓN: 🟡 REVISAR (está entre 30-85 puntos)
Analogía:

Es como un detector de metales en el aeropuerto. Pitas un poco = revisión rápida. Pitas mucho = revisión exhaustiva. Pitas DEMASIADO = problema serio y seguridad interviene.

🔍 Sistema de Detección Vectorial

¡LA MAGIA! El corazón del sistema completo

def detectar_fraude_vectorial(transaccion):
# PASO 1: Generar embedding
embedding_txn = model.encode(descripcion_txn)

# PASO 2: Vector Search
vecinos = db.customers.aggregate([{
"$vectorSearch": {
"queryVector": embedding_txn,
"limit": 3
}
}])

# PASO 3: Calcular risk score
score = calcular_risk_score(transaccion)

# PASO 4: Penalización por baja similitud
if similitud < 0.5:
score += 40

# PASO 5: Decisión final
return decision, score

🚀 ¿Qué hace?

Este es el corazón del sistema. Combina Vector Search con Risk Scoring para detectar fraudes de manera inteligente.

🎯 Flujo Completo de Detección

PASO 1: Convertir en Vector "$5M en casino a las 3 AM" → [0.89, -0.45, 0.23, ... 384 números]
⬇️
PASO 2: Vector Search MongoDB busca: "¿Qué clientes tienen comportamiento similar?"
Resultado: Cliente Alejandro (gerente) - Similitud: 0.3 (BAJA ⚠️)
⬇️
PASO 3: Calcular Risk Score • Monto 50x el promedio → +30 puntos
• Hora 3 AM → +15 puntos
• Dispositivo nuevo → +35 puntos
Subtotal = 80 puntos
⬇️
PASO 4: Penalización Vectorial Similitud 0.3 es MUY baja (no se parece a nadie conocido) → +40 puntos extra
Score Final = 80 + 40 = 120 puntos 🚨
⬇️
PASO 5: Decisión Final
120 > 85 → 🔴 BLOQUEAR
Analogía:

Es como la seguridad del aeropuerto con dos capas:

  • Vector Search = Reconocimiento facial que compara tu cara con perfiles conocidos
  • Risk Model = Detector de metales que suma puntos por items sospechosos
  • Si ambos sistemas dicen "sospechoso" → ¡ALARMA MÁXIMA!

Métrica Qué Mide Importancia
True Positives (TP) Fraudes detectados correctamente ✅ ¡Dinero salvado!
False Negatives (FN) Fraudes que dejamos pasar ❌ Pérdida de dinero
False Positives (FP) Clientes legítimos bloqueados ⚠️ Clientes molestos
Precisión % de bloqueados que son fraudes reales TP / (TP + FP)
Recall % de fraudes reales que detectamos TP / (TP + FN)
Dinero Salvado Suma de montos de fraudes bloqueados 💰 ROI del sistema
