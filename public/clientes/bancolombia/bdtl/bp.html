Recursos esenciales para la implementación y optimización de MongoDB en el proyecto BDTL de Bancolombia.

Fundamentos para el Éxito

Migración estratégica de SQL Server a MongoDB

Esta guía reúne las mejores prácticas, patrones de diseño y recursos oficiales de MongoDB para garantizar una migración exitosa del sistema BDTL. Cada recurso ha sido seleccionado específicamente para abordar los desafíos de transformar una arquitectura relacional a un modelo orientado a documentos.

🎯 Guías Técnicas Especializadas

🔍

Wildcard Indexes

Índices dinámicos para campos variables

Fundamentales para indexar documentos con esquemas flexibles como los datos líquidos de BDTL. Permiten crear índices automáticamente en campos que pueden variar entre documentos.

📚 Ver Documentación
⚡

Compound Wildcard Indexes

Índices compuestos avanzados

Combinan índices wildcard con campos específicos para optimizar consultas complejas. Ideales para consultas que filtran por campos fijos (como EntidadId) y campos variables dentro de DatosLiquidos.

🔧 Explorar Técnica
🎯

Partial Indexes

Índices condicionalmente eficientes

Indexan únicamente documentos que cumplen condiciones específicas, reduciendo el tamaño del índice y mejorando el rendimiento. Perfectos para indexar solo documentos activos o de ciertos tipos.

🧩 Patrones Implementados en BDTL

🔗

Extended Reference Pattern

Incrusta campos frecuentemente consultados de documentos referenciados para eliminar JOINs. Aplicado en la desnormalización de entidades, canales y tecnologías.

🕒

Document Versioning

Gestiona cambios en documentos manteniendo versiones. Útil para el control de auditoría en registros BDTL que requieren trazabilidad histórica.

📊

Outlier Pattern

Separa documentos que crecen excesivamente. Aplicado en la separación de logs de auditoría para mantener el documento principal compacto.

⏱️

Schema Versioning

Permite evolución gradual del esquema sin migración masiva. Facilita actualizaciones incrementales del modelo de datos BDTL.

🎯

Computed Pattern

Pre-calcula valores derivados durante la escritura. Implementado en el cálculo de FechaVigencia y FechaDepuracion en el microservicio de ingesta.

🗂️

Bucket Pattern

Agrupa documentos relacionados en "buckets" para optimizar consultas. Considerado para agrupar registros por período temporal en consultas de batch.

✅ Mejores Prácticas para BDTL

📐

Diseño Orientado a Consultas

Modela los datos basándote en cómo se van a consultar, no en cómo se almacenan en SQL. BDTL prioriza velocidad de lectura sobre normalización perfecta.

🔍

Índices Estratégicos

Crea índices compuestos que cubran múltiples campos de consulta. Usa wildcard indexes para datos líquidos variables y partial indexes para filtros condicionales.

⚡

Desnormalización Inteligente

Incrusta datos frecuentemente consultados para eliminar JOINs. Acepta duplicación controlada a cambio de performance excepcional en lecturas.

📏

Documentos de Tamaño Predecible

Mantén documentos bajo 16MB y evita arrays que crezcan sin límite. Separa logs y datos grandes en colecciones independientes.

🔄

Operaciones Atómicas

Aprovecha que las operaciones a nivel de documento son atómicas. Agrupa datos relacionados para garantizar consistencia sin transacciones complejas.

⏰

TTL para Datos Temporales

Usa índices TTL (Time To Live) para automatizar la depuración de datos. Reemplaza jobs complejos de limpieza con expiración automática.

🧪

Evolución de Esquema Gradual

Diseña documentos que puedan evolucionar sin migración masiva. Usa campos opcionales y versioning para cambios incrementales.

📊

Aggregation Framework

Usa pipelines de agregación para análisis complejos en lugar de traer datos a la aplicación. Procesa datos donde residen para mejor performance.
