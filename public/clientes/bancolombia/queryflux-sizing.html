1. Análisis de Carga de Trabajo

Desglosando el comportamiento de los datos para justificar el hardware.

Teoría del "Working Set"

Aunque el volumen total es de 542 GB, en sistemas transaccionales de fraude, los datos accedidos frecuentemente (Hot Data) rara vez superan el 20%.

Total Data (Raw): 542 GB
Compresión WiredTiger (~50%): ~271 GB (en disco)
Working Set Est. (20%): ~54 GB

*Para mantener latencias < 200ms, este "Working Set" debe residir mayoritariamente en RAM.

Perfil de Operaciones (900 TPS)

Lecturas (Fraud Checks) 70%
Escrituras (Updates/Inserts) 30%

Implicación: Necesitamos gestión de IOPS eficiente. Sin NVMe, la contención de disco será un riesgo mayor.

2. Cálculo de Hardware: La Batalla de RAM

Análisis comparativo de memoria para M50 vs M60.

OPCIÓN DESACOPLADA

El Enfoque M50

El M50 ofrece 32GB. Al usar Nodos de Search externos, liberamos esta RAM.

  • Total RAM: 32 GB
  • OS + Overhead: -4 GB
  • Cache Datos: ~28 GB

Toda la memoria se dedica a transacciones. Los índices de búsqueda viven fuera.

OPCIÓN CONSOLIDADA

M60 (Standard SSD)

64GB RAM con disco estándar. Ideal para ahorrar costos, riesgoso para I/O.

  • Total RAM: 64 GB
  • OS + Overhead: -6 GB
  • Search Index: -15 GB
  • Cache Datos: ~43 GB

Advertencia: Al no ser NVMe, el throughput de disco es compartido.

Comparativa de Arquitecturas

OPCIÓN ECONÓMICA

Cluster M60 Standard

64GB RAM | 16 vCPUs | Standard I/O

Arquitectura consolidada. Eliminamos la capacidad NVMe para reducir costos, centralizando todo en un cluster de propósito general.

Implicaciones Técnicas:

  • IOPS Contention: El mayor riesgo. Las 900 TPS y la indexación de búsqueda compiten por los mismos IOPS limitados del disco estándar.
  • Gestión Unificada: Sigue siendo un solo cluster, simplificando la operación.
  • Latencia Variable: Sin NVMe, es difícil garantizar < 200ms consistentes durante picos de carga.
Requisito Crítico Vector Quantization ACTIVADO
Recomendada

M50 + Search Nodes

32GB + 2x32GB Dedicated | Aislamiento Total

Arquitectura de microservicios físicos. Separamos la carga transaccional (pagos/fraude) de la carga analítica (búsqueda de patrones).

Ventajas de Arquitectura:

  • SLA Blindado: El aislamiento de hardware protege las transacciones de los picos de I/O de búsqueda.
  • Precisión Máxima: Gracias a la RAM dedicada en los nodos S30, podemos usar vectores puros (float32).
  • Escalado Independiente: ¿Más búsquedas? Agrega nodos S30. ¿Más transacciones? Sube el M50.

Configuración Final Sugerida

CLUSTER PRINCIPAL M50 General 3 Nodos Replica Set
VECTOR SEARCH 2x S30 Nodes Dedicados NVMe
ESTRATEGIA DE ESCALADO

Inicio en M50 para validar "Working Set". Si la retención de datos calientes aumenta >60 días, escalado vertical transparente a M60 sin downtime. Quantization se mantiene como "carta bajo la manga" para optimización futura de costos.
