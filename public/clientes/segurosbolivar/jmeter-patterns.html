Recomendado

1. The Agent Sidecar

Desacoplamiento total usando un agente ligero (Telegraf) como buffer intermedio.

JMeter
Graphite (UDP)
Telegraf (Agent)
Bulk Write
MongoDB Atlas
*JMeter "dispara y olvida". Telegraf gestiona la red.*
  • ✓ Zero Code en Java
  • ✓ Buffer anti-bloqueo
  • ✓ Estándar de la industria

Esfuerzo

Bajo (Configuración)

Alto Control

2. Custom Listener

Implementación nativa en Java dentro del proceso de JMeter.

JMeter
Java Plugin
Direct TCP
MongoDB Atlas
*El plugin comparte recursos (CPU/RAM) con la prueba.*
  • ✓ Transformación compleja
  • ✕ Riesgo de bloqueo de threads
  • ✕ Alto mantenimiento (Code)

Esfuerzo

Alto (Desarrollo)

Alternativa

3. Log Tailing

JMeter escribe en disco (CSV/JTL) y un agente lee el archivo.

JMeter
Write Disk
results.jtl
Read (Tail)
MongoDB Atlas
*Introduce latencia en la visualización (No es Real-Time puro).*
  • ✓ Muy seguro (File System)
  • ✕ Alta latencia (Disk I/O)
  • ! Gestión de rotación de logs

Esfuerzo

Medio (Configuración)

Matriz de Decisión Técnica

Criterio 1. Telegraf (Agent) 2. Custom Java 3. Log Tailing
Impacto en JMeter Mínimo (Fire & Forget) Alto (Bloqueante) Medio (Disk I/O)
Tiempo Real Excelente (<1s) Excelente (Inmediato) Con retraso (Lag)
Mantenimiento Configuración (TOML) Código (Java Compilado) Configuración (Regex)
Robustez de Red Alta (Buffer Local) Baja (Fallo directo) Alta (Persistencia en disco)
