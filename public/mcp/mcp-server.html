01 — Protocolo

Que es el Model Context Protocol

MCP es un estandar abierto creado por Anthropic que permite a los modelos de lenguaje (LLMs) comunicarse con fuentes de datos externas de forma segura y estructurada. Piensa en MCP como el "USB-C" de la inteligencia artificial: un conector universal entre tu AI y tus herramientas.

"El MCP Server no le da inteligencia a la base de datos —
le da contexto a la inteligencia artificial"

02 — Arquitectura

Dos planos, un servidor

El MCP Server opera en dos planos complementarios: el plano de datos para operaciones sobre tu base de datos, y el plano de control para gestionar tu infraestructura Atlas.

Plano de Datos

Conexion directa a tu base de datos MongoDB a traves del connection string. Permite ejecutar queries, aggregations, crear indices, explorar esquemas y generar embeddings.

Requiere: connectionString
Protocolo: MongoDB Wire Protocol
Acceso: Base de datos, colecciones, documentos

Plano de Control

Interactua con la Atlas Administration API para gestionar clusters, usuarios, alertas, logs y proyectos. Requiere API Keys de Atlas con los permisos apropiados.

Requiere: atlas.publicKey + atlas.privateKey
Protocolo: HTTPS / REST API
Acceso: Clusters, proyectos, usuarios, alertas

Herramientas Destacadas
aggregate

Ejecuta pipelines de agregacion completos. Soporta $match, $group, $lookup, $unwind, $project y todos los stages del framework de agregacion.

collectionSchema

Analiza una muestra de documentos para inferir el esquema de la coleccion: tipos de datos, campos opcionales, arrays anidados y sub-documentos.

explain

Devuelve el plan de ejecucion de una consulta. Identifica si se estan usando indices, muestra el numero de documentos escaneados vs retornados.

createEmbedding

Genera embeddings vectoriales usando modelos configurados. Permite transformar texto en vectores para busquedas semanticas con Atlas Vector Search.

atlas-inspect-cluster

Detalle completo del cluster: tier, version MongoDB, region, storage, estado de backup, conexiones activas y metricas basicas.

atlas-logs

Recupera logs del proceso mongod/mongos para diagnostico. Util para identificar queries lentas, errores de autenticacion y problemas operativos.

05 — Seguridad

Seguridad por diseno

El MCP Server incluye multiples capas de proteccion para garantizar que tu base de datos y tu infraestructura esten protegidas, incluso cuando un modelo de AI interactua con ellas.

Modo Solo Lectura

Activa --readOnly para deshabilitar todas las operaciones de escritura. Ideal para entornos de produccion donde la AI solo debe consultar.

--readOnly

Herramientas Deshabilitables

Controla granularmente que herramientas estan disponibles. Deshabilita operaciones destructivas o sensibles segun tu politica de seguridad.

--disabledTools "dropCollection,
deleteMany,dropIndex"

Control de Telemetria

La telemetria es completamente opcional. Desactivala para cumplir con politicas de privacidad corporativas. Ningun dato se envia sin consentimiento.

--telemetry disabled

Verificacion de Indices

Cuando esta habilitado, el MCP Server verifica que existan indices apropiados antes de ejecutar consultas. Previene escaneos completos de coleccion.

--indexCheck

RBAC de MongoDB

Los permisos del MCP Server estan limitados por los roles del usuario de base de datos. Si el usuario es readOnly en MongoDB, el MCP Server tambien lo sera.

Sin Almacenamiento Local

El MCP Server no almacena credenciales, queries ni resultados en disco. Todo se maneja en memoria durante la sesion y se descarta al terminar.

08 — Performance Advisor

Optimizacion automatica de rendimiento

La integracion con Atlas Performance Advisor permite que tu asistente de AI identifique queries lentas y recomiende indices de forma proactiva.

1

Deteccion

El Performance Advisor analiza tu workload y detecta queries sin cobertura de indice.

2

Analisis

El MCP Server expone las recomendaciones al agente AI con contexto de impacto.

3

Recomendacion

El agente sugiere los indices a crear con explain() del antes y despues.

4

Ejecucion

Con tu aprobacion, el agente ejecuta createIndex() y verifica la mejora.

09 — Casos de Uso

Escenarios reales para Bancolombia

Estos son escenarios concretos donde el MCP Server puede acelerar el trabajo de los equipos de desarrollo y operaciones del banco.

Diagnostico del Query Model

"Analiza el esquema de MovementsEndOfDay y recomienda indices para las consultas mas frecuentes." El agente inspecciona la coleccion, lista indices existentes y sugiere optimizaciones.

Auditoria de Indices No Utilizados

"Identifica indices que no se estan usando en produccion." El agente cruza el Performance Advisor con listIndexes para detectar los 35GB en indices redundantes.

Generacion de Codigo para Bonos

"Genera el modelo de datos y validaciones para el nuevo producto Bonos." El agente infiere el esquema desde documentos existentes y genera el codigo Node.js/Java completo.

Monitoreo Proactivo de Clusters

"Como esta el cluster M80 de produccion?" El agente consulta metricas, alertas, uso de disco (358GB/598GB), cache de WiredTiger (80%) y lock contention (25.7%).

Consultas Ad-hoc en Tiempo Real

"Cuantas cuentas de depositos se actualizaron hoy?" El agente construye y ejecuta la query sobre MovementsRealtime sin que el desarrollador escriba una linea de codigo.

Gestion de Accesos

"Crea un usuario readOnly para el equipo de conciliacion con acceso solo a la coleccion snapshots_saldos." Gestion granular de permisos desde lenguaje natural.
